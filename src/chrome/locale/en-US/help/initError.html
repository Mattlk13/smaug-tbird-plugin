How to Resolve Problems with Initializing Smaug

There are several reasons why initializing Smaug does not succeed. The most common ones are described below; for more information please visit the Smaug Support page.

GnuPG could not be found

In order for Smaug to work, the tool GnuPG needs to be installed. If GnuPG cannot be found, then first make sure that the executable gpg.exe (on Windows; gpg on other platforms) is installed on your computer. If GnuPG is installed, and Smaug cannot find it, then you need to manually set the path to GnuPG in the Smaug Preferences (menu Smaug > Preferences)

Smgmime failed to initialize

Smaug works only if it is built using the same build environment as Thunderbird or SeaMonkey was built. This means that you can use the official Smaug releases only if you use the official releases of Thunderbird or SeaMonkey provided by mozilla.org.

If you use a Thunderbird or SeaMonkey version coming from some other source (e.g. the provider of your Linux distribution), or if you built the application yourself, you should either use an Smaug version built by the same source, or build Smaug yourself. For building Smaug, refer to the Source Code section on the Smaug home page. Please don't file any bug report concerning this problem, it is not solvable.

Further help is available on the Smaug Support Web Site.
